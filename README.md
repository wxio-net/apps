# YouliYouli 安装与使用说明

本文档对应应用 **YouliYouli**（图形界面视频下载工具）。若你从 `.zip` 或网盘取得打包版本，请按下列步骤安装与授权。

---

## 一、应用简介

YouliYouli 是一款在 **macOS** 上运行的桌面程序，通过内置的 **yt-dlp**、**ffmpeg**、**Node** 等组件，从网络获取视频流并保存到本地。应用免费使用，体积较大是因为自带运行所需的可执行文件与依赖。

- 官网 / 说明页：见应用内 **关于** 窗口中的链接（与源码中 `AUTHOR_URL` 一致）。
- 反馈邮箱：见 **关于** 窗口（`seofou@gmail.com`）。

使用前请自行遵守视频所在平台的**服务条款**与**版权法律**，仅下载你有权获取的内容。

---

## 二、主要功能

| 功能 | 说明 |
|------|------|
| **画质** | 可选高清 720p / 全高清 1080p（H.264 倾向）。 |
| **字幕** | 可选内嵌字幕（自动字幕多语言，含中英相关语言代码）。 |
| **缩略图** | 可选内嵌封面图（若源站提供）。 |
| **保存位置** | 默认用户「下载」文件夹，可浏览修改，并支持在 Finder 中打开。 |
| **批量地址** | 主界面大文本框内**每行一个**链接；开始下载前会记住列表。 |
| **开始 / 停止** | 同一按钮切换：下载进行中显示为「停止」。 |
| **高级参数** | 一行内填写额外传给 yt-dlp 的参数（空格分隔，含引号时需成对，规则与终端一致）。 |
| **网络测试** | 使用本机 `curl` 探测访问 YouTube 的连通性；若你在高级参数里填写了 `--proxy`，测试会尝试使用该代理。 |
| **浏览器 Cookie** | 当**高级参数里没有** `--cookies-from-browser` 时，会自动为 yt-dlp 追加 **`--cookies-from-browser`**，浏览器名称由系统默认 HTTPS 浏览器推断（详见下文「权限与 Cookie」）。 |
| **配置存盘** | 设置保存在用户主目录 **`~/.youliyouli/config.conf`**（含画质、目录、高级参数、视频列表等）。 |
| **关于** | 版本信息、可点击的邮箱与网页链接、简要说明文案。 |

---

## 三、系统与环境要求

- **操作系统**：当前发行版面向 **macOS**（应用内调用 `yt-dlp_macos` 及同目录下的 `ffmpeg`、`node`）。
- **可选**：终端需能使用 `curl`（用于「测试网络」，一般为系统自带）。
- **磁盘空间**：除下载文件外，请预留足够空间存放应用包与临时文件。

## 四、安装步骤（从压缩包安装）

1. **解压**  
   若收到的是 `YouliYouli.zip`，双击解压后会得到 **`YouliYouli.app`**。

2. **移入「应用程序」（推荐）**  
   将 `YouliYouli.app` 拖入 **访达 → 应用程序** 文件夹，便于启动与统一管理。

3. **首次打开**  
   - 若系统提示 **来自身份不明的开发者** 或无法打开，请先完成下文 **「解除隔离（xattr）」**。  
   - 也可尝试：**系统设置 → 隐私与安全性** 中，对本次拦截选择 **仍要打开**（具体文案因 macOS 版本可能略有不同）。

4. **可选：解除隔离命令**  
   若提示应用**已损坏**或始终无法打开，在 **终端** 执行（路径请改为你实际放置 `.app` 的位置）：

   ```bash
   xattr -cr "/Applications/YouliYouli.app"
   ```

   若应用放在「下载」里，例如：

   ```bash
   xattr -cr ~/Downloads/YouliYouli.app
   ```

   **说明**：从互联网下载的 `.app` 会带 **隔离标记（quarantine）**，`xattr -cr` 会递归清除扩展属性中的隔离标记，多数情况下可恢复正常双击打开。**不会**修改应用内部业务逻辑。

5. **开发者签名（可选）**  
   若你持有 **Apple 开发者证书**，可自行对整个 `.app` 签名或公证；未签名/未公证的包在他人电脑上更容易出现 Gatekeeper 拦截，可配合说明文档提供上述 `xattr` 步骤。

## 五、权限与「读取浏览器」（Cookie）

为使需要登录态的网站能正常解析，应用在**未**于高级参数中手动写 `--cookies-from-browser` 时，会自动注入 **yt-dlp 的** `--cookies-from-browser` **及推断出的浏览器名**（如 chrome、safari、edge、firefox 等，与 yt-dlp 要求一致）。

### 5.1 应用自身在做什么

- **macOS**：通过读取本机 **`com.apple.launchservices.secure.plist`** 等逻辑推断**默认处理 HTTPS 的浏览器**，再映射为 yt-dlp 支持的浏览器名称。
- **真正读 Cookie / 登录态** 的是 **yt-dlp**（含其依赖），会去访问对应浏览器的用户数据目录或密钥链（视浏览器而定）。

### 5.2 你可能看到的系统行为

- **首次或偶发弹窗**：系统可能询问是否允许终端/脚本访问**文件**、**钥匙串**或某浏览器的数据（与 macOS 版本及浏览器有关）。
- **完整磁盘访问权限**：若 Safari 等数据受系统保护，必要时可在 **系统设置 → 隐私与安全性 → 完整磁盘访问权限** 中为所需程序勾选（一般优先保证 **终端** 或 **YouliYouli** 若单独列出的条目，以你机器上实际提示为准）。
- **浏览器请尽量完全退出再试**：部分浏览器的配置数据库在运行时锁定，可能导致 yt-dlp 无法读取 Cookie；可退出浏览器后重试下载。

### 5.3 自行控制 Cookie 行为

在 **高级参数** 中你可手动添加 yt-dlp 支持的形式，例如：

- `--cookies-from-browser chrome`（指定浏览器，会覆盖自动推断逻辑）
- `--cookies /path/to/cookies.txt`（使用导出文件）

若高级参数里**已经**包含 **`--cookies-from-browser`**（含 `=` 形式），应用**不会**再自动追加一遍。

## 六、使用简要说明

1. 在地址框中输入或粘贴链接（每行一个），点击 **开始下载**。  
2. 下载过程中同一按钮变为 **停止**，可随时中断队列。  
3. 多个地址会按顺序下载，每条任务之间约有短暂间隔（减轻对源站压力）。  
4. 状态区显示 yt-dlp 输出；新一轮下载开始时会清空上一轮的日志。  
5. 修改画质、目录、高级参数后，在开始下载时会写入 **`~/.youliyouli/config.conf`**。

---

## 七、配置文件位置

| 项目 | 路径 |
|------|------|
| 配置文件 | `~/.youliyouli/config.conf` |

格式为简单的 `键=值` 文本；可备份该文件以迁移设置。若文件损坏，可关闭应用后删除该文件再启动（会丢失已保存选项）。

## 八、常见问题（简要）

- **提示损坏 / 无法打开**：先执行 **`xattr -cr` 指向你的 `.app` 路径**（见第四节）。  
- **需要登录的视频失败**：检查浏览器 Cookie 权限；或在高级参数中显式指定 `--cookies-from-browser` / `--cookies`。  
- **高级参数不生效**：检查引号是否成对；错误语法会在状态区出现 **WARNING** 提示。  
- **代理**：在高级参数中加入 yt-dlp 支持的代理参数（如 `--proxy http://127.0.0.1:7890`）；「测试网络」会尝试从该行中解析 `--proxy` 或 `--proxy=`。
